// Overview

W1seGuy is a TryHackMe room that simulates a service running an XOR encryption challenge. The server gives an XOR-encrypted version of a flag and asks for the encryption key. When you submit the correct key, it reveals the second flag.

The room is a hands-on exercise in cryptanalysis. With the encrypted data and knowledge of the flag format, we can find the encryption key without resorting to brute force. This shows why XOR by itself is not secure encryption and highlights how known-plaintext attacks can threaten weak cryptographic implementations.

// Background: XOR & Known-Plaintext Attacks

XOR (exclusive OR) is a bitwise operation with a unique property: if A XOR B equals C, then C XOR B equals A. Encryption with XOR is symmetric; the same operation is used for both encryption and decryption.

When the same key is repeated (a repeating-key XOR), patterns start to show. If an attacker knows any part of the plaintext, they can find the corresponding key bytes by XORing the ciphertext with the known plaintext.

In this challenge, the flag format THM{...} gives the known plaintext needed to recover the entire key.

// Reconnaissance: Examining the Source

The room provides a Python script as a downloadable attachment. Let's take a look at the key sections.

flag = open('flag.txt','r').read().strip()

def setup(server, key):
    flag = 'THM{thisisafakeflag}'  # placeholder
    xored = ""
    for i in range(0,len(flag)):
        xored += chr(ord(flag[i]) ^ ord(key[i%len(key)]))
    hex_encoded = xored.encode().hex()
    return hex_encoded

def start(server):
    key = ''.join(random.choices(string.ascii_letters + string.digits, k=5))
    hex_encoded = setup(server, key)
    send_message(server, "This XOR encoded text has flag 1: " + hex_encoded + "\n")
    key_answer = server.recv(4096).decode().strip()
    if key_answer == key:
        send_message(server, "Congrats! ... Here is flag 2: " + flag + "\n")

From this, we can see:

  • The flag format is THM{...} (the placeholder confirms this).
  • The key is 5 characters long and alphanumeric.
  • Each character of the flag is XORed with a repeating key character.
  • The result is hex-encoded and sent to the client.

The server runs this script, generates a random key for each connection, encrypts the flag, and asks for the key. Our job is to recover the key from the encrypted data we receive.

// The Attack: Known-Plaintext XOR Key Recovery

The encryption process involves three steps:

  1. Generate a random 5-character alphanumeric key.
  2. XOR each character of the flag with a repeating key character.
  3. Hex-encode the result and send it to the client.

The weakness lies in the fact that we know part of the plaintext, which follows the format THM{...}.

Since XOR is symmetric (A ⊕ B = C and C ⊕ B = A), we can recover the key if we have both the ciphertext and the plaintext.

  • First 4 characters: We know the plaintext starts with THM{. XORing these with the first 4 bytes of the ciphertext reveals the first 4 characters of the key.
  • 5th character: Since the flag length is a multiple of 5, the last character of the plaintext (}) was XORed with the 5th character of the key. XORing these provides the last key character.

// Execution: Connecting & Decrypting

Step 1: Capture the Ciphertext

First, connect to the service to get the encrypted flag. The server gives you a hex string.

jackhacksstuff@kali:~$ nc <target_ip> 1337
This XOR encoded text has flag 1: <hex_string>
What is the encryption key?

Copy the <hex_string>. This is our ciphertext.

Step 2: Understand the XOR Math

From the source code, we know:

  • Encryption: Ciphertext = Plaintext ⊕ Key
  • Decryption: Plaintext = Ciphertext ⊕ Key
  • XOR is its own inverse. If we know the plaintext and the ciphertext, we can recover the key: Key = Ciphertext ⊕ Plaintext.

We also have two important pieces of plaintext:

  • The flag format: It starts with THM{ and ends with }.
  • The key length is 5 characters.

Step 3: Recover the Key

Let's use Python to recover the key step by step.

import binascii

# Paste your hex string here
hex_encoded = "<hex_string>"
ciphertext = bytes.fromhex(hex_encoded)

# 1. Recover the first 4 key characters from the known start "THM{"
known_start = b"THM{"
key_start = bytes([ciphertext[i] ^ known_start[i] for i in range(4)])
print(f"First 4 key chars: {key_start.decode()}")

# 2. Recover the 5th key character from the known end "}"
key_end = bytes([ciphertext[-1] ^ ord('}')])
print(f"5th key char: {key_end.decode()}")

# 3. Combine to get the full key
key = key_start + key_end
print(f"Full key: {key.decode()}")

# 4. Decrypt the flag using the recovered key
flag = bytes([ciphertext[i] ^ key[i % len(key)] for i in range(len(ciphertext))])
print(f"Flag 1: {flag.decode()}")

Pro Tip: Using pwntools
For a simpler approach, you can use the pwntools library. It handles the connection and XOR operations smoothly. Here's a script that does everything in one go: 
from pwn import *

# Connect to the target
conn = remote('<target_ip>', 1337)

# Receive the hex string
data = conn.recvuntil(b'What is the encryption key?')
hex_encoded = data.split(b'flag 1: ')[1].split(b'\n')[0].decode()

ciphertext = bytes.fromhex(hex_encoded)
known_start = b"THM{"

# Recover the key
key_start = xor(ciphertext[:4], known_start)
key_end = xor(ciphertext[-1:], b'}')
key = key_start + key_end

# Decrypt the flag
flag1 = xor(ciphertext, key)

print(f"Recovered key: {key.decode()}")
print(f"Flag 1: {flag1.decode()}")

# Send the key to get flag 2
conn.sendline(key)
response = conn.recvall()
print(response.decode())
conn.close()

// Key Takeaways

XOR is not encryption. It is a bitwise operation, not a method for protecting secrets. Without a truly random, non-repeating key (a one-time pad), XOR-based "encryption" is weaks against attacks.
Known-plaintext attacks are a real threat. This challenge shows that knowing even a small part of the plaintext (the flag format) can expose the entire encrypted message.
Key length is important. A 5-character key made from 62 possible characters creates about 916 million possibilities, which is easy for modern hardware to break. Even without the known-plaintext shortcut, brute force would still be possible.
Always handle binary data as bytes. When using XOR and hex-encoded data, rely on bytes and bytearray in Python. String operations can give unexpected results with null bytes or control characters.
Understand the source code. The room provided the full server source. Reading it closely revealed the key length, the encryption method, and the flag format, which are all crucial for the attack.
